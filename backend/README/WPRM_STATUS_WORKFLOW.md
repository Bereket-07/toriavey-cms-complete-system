# WPRM Content Status Workflow

## 📊 Complete Status Flow

```
not_generated → generated → pending → posted
                    ↓
                declined
                    ↓
                 failed
```

---

## 🔄 Status Definitions

### **1. `not_generated`**
- No content created yet
- Default status for all recipes
- Scheduler will pick these recipes for generation

### **2. `generated`**
- Content has been generated by LLM
- **Waiting for approval**
- Content stored in database
- **Action needed:** Review and approve/decline

### **3. `pending`**
- Content has been **approved**
- **Ready to be posted** to social media
- Waiting for posting scheduler or manual post

### **4. `posted`**
- Successfully posted to social media
- Final status
- Tracks which platforms it was posted to

### **5. `declined`**
- Content was rejected/declined
- Will be regenerated in next batch
- Treated same as `not_generated`

### **6. `failed`**
- Generation or posting failed
- Tracks error message
- Can be retried

## 🎯 Workflow Steps

### **Step 1: Generation**
```http
POST /api/wprm-scheduler/run-now
```
**Result:** Recipes move from `not_generated` → `generated`

---

### **Step 2: Review Generated Content**
```http
GET /api/content/wprm-recipes-generated-not-posted?limit=10
```

**Response:**
```json
{
  "success": true,
  "total_awaiting_approval": 5,
  "recipes": [
    {
      "id": 43615,
      "title": "Butterscotch Pie",
      "content_status": {
        "status": "generated",
        "content_generated": true,
        "posted": false
      },
      "generated_content": {
        "instagram": {...},
        "tiktok": {...},
        ...
      }
    }
  ]
}
```

---

### **Step 3A: Approve Content**
```http
POST /api/content/wprm-approve-content/43615
```

**Response:**
```json
{
  "success": true,
  "message": "Recipe 43615 approved and marked as pending",
  "recipe_id": 43615,
  "new_status": "pending"
}
```

**Result:** Recipe moves from `generated` → `pending`

---

### **Step 3B: Decline Content (Optional)**
```http
POST /api/content/wprm-decline-content/43615?reason=Poor quality caption
```

**Response:**
```json
{
  "success": true,
  "message": "Recipe 43615 content declined",
  "recipe_id": 43615,
  "new_status": "declined",
  "reason": "Poor quality caption"
}
```

**Result:** Recipe moves from `generated` → `declined` (will be regenerated)

---

### **Step 4: View Pending Recipes (Ready to Post)**
```http
GET /api/content/wprm-recipes-pending?limit=10
```

**Response:**
```json
{
  "success": true,
  "total_pending": 3,
  "recipes": [
    {
      "id": 43615,
      "title": "Butterscotch Pie",
      "content_status": {
        "status": "pending",
        "content_generated": true,
        "posted": false
      },
      "generated_content": {
        "instagram": {...},
        "tiktok": {...},
        ...
      }
    }
  ]
}
```

---

### **Step 5: Post to Social Media**

## 📈 Status Summary

```http
GET /api/content/wprm-status-summary
```

**Response:**
```json
{
  "success": true,
  "total_recipes": 729,
  "not_generated": 720,
  "generated": 5,
  "pending": 3,
  "posted": 1,
  "failed": 0,
  "completion_percentage": 0.14,
  "generation_percentage": 1.23
}
```

**Metrics:**
- `not_generated`: Need content generation
- `generated`: Awaiting approval
- `pending`: Approved, ready to post
- `posted`: Successfully posted
- `failed`: Errors
- `completion_percentage`: % posted
- `generation_percentage`: % with content (generated + pending + posted)

## 🔌 All Status Endpoints

### **Get Recipes by Status:**

1. **Not Generated** (need generation)
   ```http
   GET /api/content/wprm-recipes-not-generated
   ```

2. **Generated** (need approval)
   ```http
   GET /api/content/wprm-recipes-generated-not-posted
   ```

3. **Pending** (ready to post)
   ```http
   GET /api/content/wprm-recipes-pending
   ```

### **Status Actions:**

1. **Approve Content**
   ```http
   POST /api/content/wprm-approve-content/{recipe_id}
   ```

2. **Decline Content**
   ```http
   POST /api/content/wprm-decline-content/{recipe_id}
   ```

3. **Get Specific Recipe Status**
   ```http
   GET /api/content/wprm-recipe-status/{recipe_id}
   ```

## 💡 Use Cases

### **Automated Generation + Manual Approval**
1. Scheduler generates content every hour
2. You review and approve/decline
3. Approved content goes to `pending`
4. Post pending content to social media

### **Batch Approval**
1. Generate 20 recipes
2. Review all generated content
3. Approve 15, decline 5
4. Post all 15 pending recipes

### **Quality Control**
1. Generate content
2. Review quality
3. Decline poor quality → will be regenerated
4. Approve high quality → ready to post

## 📊 Database Tracking

All status changes are tracked in `wprm_content_status` table:

```sql
SELECT 
  recipe_id,
  status,
  content_generated,
  generation_date,
  posted,
  post_date,
  platforms_posted,
  notes
FROM wprm_content_status
WHERE status = 'pending'
ORDER BY generation_date DESC;
```
